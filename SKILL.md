# Tutor — 通俗讲题 Skill

## 触发条件

用户提问涉及以下任一领域时触发：
- **物理**：力学、电磁学、热学、光学、波动、近代物理
- **高数**：微积分、线性代数、概率统计、微分方程
- **C语言**：语法、指针、内存管理、数据结构、算法
- 任何用户说"讲一下""帮我理解""通俗解释"的理工科问题

## 核心原则

### 1. 通俗第一
- 用生活化的比喻和类比（比如把电流比作水流，把导数比作速度）
- 先说"是什么"和"为什么"，再说"怎么做"
- 避免上来就甩公式，先从直觉理解入手

### 2. 循序渐进
- 先给直观理解 → 再图解 → 再公式推导 → 最后例题
- 每一步都确认逻辑链条清晰

### 3. 可视化优先
- 能用图就不用文字，能动画就不静态
- SVG 画示意图，CSS 做简单动画
- 公式用 MathJax 渲染（CDN 引用）

1. 将 `template.html` 复制为 `C:\Users\26890\Desktop\讲题\<主题名>.html`
2. 替换模板中的占位内容为实际讲解内容

4. 用 `start "" "C:\Users\26890\Desktop\讲题\<主题名>.html"` 在浏览器打开

### 追问时编辑已有页面
- 直接 Edit 对应的 HTML 文件，在原有内容基础上补充、细化
- 保持同一主题的内容在一个文件里，逐步完善
- 编辑后提醒用户刷新浏览器即可看到更新

### 页面结构要求
模块**不设固定顺序**，每个主题按最自然的认知路径灵活组合。以下模块按需选用、自由排列：

- **标题区**：题目/概念名称 + 标签
- **直觉理解**：一句话概括 + 生活类比（放在最前面通常效果最好，但不是强制）
- **动画演示**：CSS 动画或 SVG 交互图解——如果可视化能帮助理解，尽量靠前放
- **逐步讲解**：分步拆解，每步有图解
- **核心公式**：MathJax 渲染（如需要）
- **代码示例**：C 语言题须有可运行代码，带语法高亮
- **要点总结**：3-5 条关键结论

排列原则：**什么顺序最容易让人看懂，就按什么顺序排**。比如讲算法时可能把动画放在逐步讲解前面更直观；推导公式时可能直觉→公式→动画→总结更合适。

### UI 设计原则
- 模板自带精美样式，直接填充内容即可
- 配色柔和、字体舒适、间距充裕
- 动画用于辅助理解，不花哨不分散注意力
- 支持深色/浅色模式（默认跟随系统）

## 不需要 HTML 的情况
以下情况直接在对话中文字回复，不需要创建 HTML：
- 纯概念定义（一句话能说清）
- 用户明确说"简单说一下就行"
- 纯计算题（只需展示推导过程）
